## 📋 Overview

**School Management Software** is a comprehensive, feature-rich education management system built on **ERPNext v15** and the **Education module**. It extends ERPNext with 90+ custom DocTypes covering the entire school lifecycle — from admissions and academics to hostel, transport, library, events, exams, and beyond.

The application is designed for **K-12 schools** and offers an integrated suite of tools for administrators, teachers, students, and parents.

## 🚀 Installation

### Prerequisites

- **ERPNext v15** (Frappe v15)
- **Education module** installed
- Python 3.11+
- Node.js 18+

### Steps

```bash
# 1. Get the app from the bench
bench get-app https://github.com/Sudhakar1110/school_management_software.git

# 2. Install on your site
bench --site your-site.com install-app school_management_software

# 3. Run the after-migrate setup (applies fixtures)
bench --site your-site.com migrate

# 4. (Optional) Generate demo data
bench --site your-site.com execute school_management_software.demo.generate
```

### Force Demo Data Reset

```bash
# Deletes all existing demo data and regenerates from scratch
bench --site your-site.com execute school_management_software.demo.generate \
    --kwargs '{"force": true}'
```

## ⚙️ Configuration

### Required Apps

The app requires the following apps to be installed on your site:

```json
["frappe", "erpnext", "education"]
```

### Module Registration

Module registration is handled via `modules.txt`, not `hooks.py`. The app registers the **School Management Software** module, which auto-discovers:
- All DocTypes in `doctype/`
- All Reports in `report/`
- All Print Formats in `print_format/`
- All Workspaces in `workspace/`

### Fixtures

Custom Fields, Property Setters, and Client Scripts are filtered by module to prevent cross-app contamination:

```python
fixtures = [
    {"dt": "Custom Field",   "filters": [["module", "=", "School Management Software"]]},
    {"dt": "Property Setter", "filters": [["module", "=", "School Management Software"]]},
    {"dt": "Client Script",   "filters": [["module", "=", "School Management Software"]]},
    {"dt": "Module Def",      "filters": [["module_name", "=", "School Management Software"]]},
]
```

## 🧩 Architecture

The application follows the **Frappe Framework** architecture:

- **DocType-driven**: All data models are defined as JSON and auto-discovered
- **Event-driven**: 25+ document event hooks for automated workflows (fee generation, room allocation, certificate issuance, etc.)
- **Modular**: Features are organized into logical groups within the module
- **API-first**: RESTful APIs for mobile app, GPS tracking, and certificate verification
- **PWA-ready**: Progressive Web App with service worker for offline support

### Key Automation Events

| Document | Event | Action |
|---|---|---|
| Program Enrollment | `on_submit` | Auto-create sales invoice |
| Hostel Application | `on_update` | Auto-create hostel admission on approve |
| Hostel Leaves | `before_insert` | Auto-generate gate pass number |
| Book Issue | `on_submit` | Mark copy as issued |
| Book Return | `on_submit` | Release book copy |
| Student | `on_update` | Auto-create alumni record on exit |
| Hostel Check-Out | `on_submit` | Free the room |
| Grievance Box | `after_save` | Assign complaint and notify maintenance |
| School Event | `after_save` | Send notifications |
| Hall Allocation | `validate` | Validate hall capacity |
